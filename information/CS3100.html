The Greedy Framework

A greedy algorithm solves an optimization problem by making a locally optimal choice at each step to extend the current solution. It never backtracks. Greedy is not always correct — it must be proven for each problem.

Two Required Properties

Optimal Substructure: An optimal solution to the whole problem contains optimal solutions to its subproblems.

Greedy Choice Property: The greedy choice made at each step is part of some optimal solution.

⚠️ Optimal substructure alone is necessary but not sufficient. Both properties must hold.
⚠️ If greedy works → problem has optimal substructure (converse of Lemma 1).
⚠️ Optimal substructure does NOT work "the other way" — you cannot split arbitrarily and combine optimal sub-solutions.

Proof Templates

Optimal Substructure — Contradiction Template

1State: "If WHOLE is optimal, then PART is optimal for SUBPROBLEM"
2Assume for contradiction: WHOLE is optimal, but PART is NOT optimal
3Since PART is not optimal, ∃ BETTER_PART with strictly better objective value
4Construct WHOLE' = REMAINING + BETTER_PART
5Show WHOLE' has better objective value than WHOLE → contradiction
6Therefore PART must be optimal for the subproblem ∎

Greedy Choice Property — Exchange Argument Template

1State: "Greedy choice G must be in some optimal solution"
2Assume for contradiction: G ∉ optimal solution O = {o₁, o₂, …, oₙ}
3Identify element oⱼ in O that can be exchanged for G
4Show O' = O − {oⱼ} + {G} is still feasible and value(O') ≥ value(O)
5O' is optimal and contains G → contradiction ∎

Coin Change

Given denominations D = {d₁, d₂, …, dₖ} and target A cents, find the minimum number of coins summing to A.

Algorithm
denom = [25, 10, 5, 1]  // sorted descending
makeChange(cents):
    for each coin in denom:
        while coin <= cents:
            sol.append(coin)
            cents -= coin
    return sol

Runtime Θ(A / min_denomination) — linear in number of coins produced

When Greedy Fails

Denominations {1, 6, 10, 25}, A = 12:

Greedy: 10+1+1 = 3 coins. Optimal: 6+6 = 2 coins.

The proof fails at the dime case: two 6¢ coins cannot be replaced by one 10¢ coin. The greedy choice property does NOT hold for this denomination set.

Dijkstra's Algorithm

Single-Source Shortest Paths Given a weighted graph with no negative edge weights and start node s, find shortest paths from s to all other nodes.

Three Groups of Nodes

Known: Shortest path fully computed, never changes. Fringe: Reachable via one edge from known, tentative distance may update. Unknown: Not yet discovered.

Algorithm (Priority Queue)
dijkstra(G, s):
    PQ.Insert(s, dist=0)
    parent[s] = NULL; dist[s] = 0

    while PQ not empty:
        v = PQ.ExtractMin()
        for each w adjacent to v:
            if w is unseen:
                dist[w] = dist[v] + wt(v,w)
                PQ.Insert(w, dist[w])
                parent[w] = v
            else if w is fringe AND dist[v]+wt(v,w) < dist[w]:
                dist[w] = dist[v] + wt(v,w)
                PQ.decreaseKey(w, dist[w])
                parent[w] = v

Runtime Θ(E log V) with binary heap  |  Θ(V² + E) with simple list

Critical: PQ Key

Dijkstra's PQ key = dist[v] + wt(v,w) → cumulative path cost from source. This is the key difference from Prim's.

Correctness Proof Sketch (Induction)

Base: s.d = 0 = δ(s,s) ✓

IH: First k known nodes have correct shortest paths.

Step: Node v becomes known with v.d = δ(s,u) + c(u,v). If a better path exists via another fringe node v', then δ(s,u') + c(u',v') + Δ < v.d. Since Δ ≥ 0 (no negative edges), δ(s,u') + c(u',v') < v.d, meaning Dijkstra's would have selected v' instead of v. Contradiction.

Prim's Algorithm

Minimum Spanning Tree Grow a single tree by always adding the cheapest edge from a known node to an unknown node.

Algorithm
prim(G, s):
    PQ.Insert(s, wt=0)
    parent[s] = NULL

    while PQ not empty:
        v = PQ.ExtractMin()
        for each w adjacent to v:
            if w is unseen:
                PQ.Insert(w, wt(v,w))    // ← just edge weight!
                parent[w] = v
            else if w is fringe AND wt(v,w) < dist[w]:
                dist[w] = wt(v,w)           // ← just edge weight!
                PQ.decreaseKey(w, dist[w])
                parent[w] = v

Runtime Θ(E log V) — identical to Dijkstra's

Critical: PQ Key

Prim's PQ key = wt(v,w) → single edge weight only. NOT cumulative distance.

Kruskal's Algorithm

Minimum Spanning Tree Grow a forest: sort all edges by weight, add cheapest edge that doesn't create a cycle.

Algorithm
kruskal(G):
    edgesAccepted = 0
    DisjointSet s(NUM_VERTICES)
    PriorityQueue Q // all edges sorted by weight

    while edgesAccepted < NUM_VERTICES - 1:
        e = Q.dequeue()       // cheapest remaining edge (u,v)
        uset = s.find(u)
        vset = s.find(v)
        if uset != vset:        // different components → no cycle
            edgesAccepted++
            s.union(uset, vset)

Runtime Θ(E log V) — heap operations dominate

Find-Union Data Structure

OperationNaïveWith Both Optimizations
makeSet(n)Θ(n)Θ(n)
find(i)O(n) worst caseΘ(α(n)) ≈ O(1) amortized
union(i, j)O(n) (find + O(1))Θ(α(n)) ≈ O(1) amortized

Optimization 1 — Union by Rank: Attach shorter tree under taller tree. Guarantees O(log n) height.

Optimization 2 — Path Compression: During find, point every node directly to root. Flattens tree for future lookups.

Together: m operations take Θ(m · α(n)) where α = inverse Ackermann ≤ 4 for all practical n.

Activity Selection

Interval Scheduling Given activities with start/finish times, select the maximum number of non-overlapping activities.

Wrong Greedy Choices

Earliest start ✗ — a long early activity blocks many shorter ones.

Shortest interval ✗ — a short interval overlapping two long non-overlapping ones gives 1 instead of 2.

Fewest conflicts ✗ — counterexample construction exists (shown in lecture).

Correct Greedy Choice

Pick the next compatible activity that finishes earliest.

Intuition: finishing early maximizes the remaining time window for future activities.

greedy_interval(s, f):  // f sorted ascending
    A = {a₁}; k = 1
    for m = 2 to n:
        if s[m] >= f[k]:   // compatible
            A = A ∪ {aₘ}; k = m
    return A

Runtime Θ(n log n) to sort + Θ(n) greedy pass. Θ(n) if pre-sorted.

Fractional Knapsack

Optimization Items with profit pᵢ and weight wᵢ. Knapsack capacity C. Can take any fraction of each item.

Wrong Greedy Choices

Highest value ✗ — ignores weight. Heavy valuable item may waste capacity.

Lowest weight ✗ — light worthless items beat heavy valuable ones.

Max leftover space ✗ — same as lowest weight. C=10, items (v=1,w=1) and (v=30,w=10): greedy gets 28, optimal is 30.

Correct Greedy Choice

Take the item with the highest value-to-weight ratio (pᵢ/wᵢ) to its maximum amount.

fractional_knapsack(items, C):
    for each item: ratio[i] = item.p / item.w
    sort(items, by ratio descending)
    weight = 0; i = 1
    while i <= n and weight < C:
        if weight + items[i].w <= C:
            take all of items[i]; weight += items[i].w
        else:
            r = (C - weight) / items[i].w
            take fraction r of items[i]; weight = C
        i++

Runtime Θ(n log n) to sort + Θ(n) pass

0/1 vs Fractional

0/1 Knapsack: Greedy FAILS. Items (v=3,w=1,r=3), (v=5,w=2,r=2.5), (v=6,w=3,r=2), C=4. Greedy by ratio → profit 8. Optimal → items 1&3 = profit 9.

Both variants have optimal substructure. The fractional proof works because you can swap continuous ε weight. 0/1 requires all-or-nothing swaps that don't fit.

Bridge Crossing

n people, one flashlight, bridge holds 2. Two cross together at slower speed. Minimize total time. Speeds sorted: s₁ ≤ s₂ ≤ … ≤ sₙ.

Strategy 1: Fastest Escorts

s₁ + sₙ cross → s₁ returns → s₁ + sₙ₋₁ cross → s₁ returns

Cost: C₁ = sₙ + s₁ + sₙ₋₁ + s₁

Strategy 2: Slowest Together

s₁ + s₂ cross → s₁ returns → sₙ₋₁ + sₙ cross → s₂ returns

Cost: C₂ = s₂ + s₁ + sₙ + s₂

The Decision Formula

C₁ − C₂ = sₙ₋₁ + s₁ − 2·s₂

If positive → use Strategy 2 (slowest together is cheaper).

If ≤ 0 → use Strategy 1 (fastest escorts is cheaper).

Recurse on remaining n − 2 people.

Runtime Deep Dive: Dijkstra's & Prim's

Why Θ(E log V)?

Both algorithms have the same structure:

• Outer loop: runs |V| times (once per vertex). Each iteration calls ExtractMin → Θ(log V).

• Inner loop: runs |E| times total across all iterations (once per edge). Each iteration may call Insert or decreaseKey → Θ(log V).

• Total: Θ(V · log V + E · log V) = Θ((V + E) · log V) = Θ(E log V) for connected graphs.

Indirect Heap: Why It Matters

decreaseKey() requires finding an element in the heap by its identity (not by position). A standard binary heap has no way to look up where a specific node lives.

Indirect heap: maintains a secondary array pos[] mapping node ID → heap index. When a node's key decreases, look up its position in O(1), then percolate up in O(log V).

Without this: decreaseKey would require O(V) linear search, making total runtime Θ(E · V).

Runtime Deep Dive: Kruskal's

Breakdown

• Sort edges: Θ(E log E). Since E ≤ V², we have log E ≤ 2 log V, so Θ(E log E) = Θ(E log V).

• Find operations: 2|E| calls (two finds per edge). With both optimizations: Θ(2E · α(V)).

• Union operations: At most |V| − 1 calls. With both optimizations: Θ(V · α(V)).

• Total: Θ(E log V + E · α(V) + V · α(V)) = Θ(E log V) since sorting dominates.

Find-Union Complexity

Configuration find() union() m operations on n elements
No optimizations O(n) O(n) O(m · n)
Union by rank only O(log n) O(log n) O(m · log n)
Path compression only amortized O(log n) amortized O(log n) O(m · log n)
Both (rank + compression) Θ(α(n)) ≈ O(1) Θ(α(n)) ≈ O(1) Θ(m · α(n))

α(n) is the inverse Ackermann function. It grows so slowly that α(n) ≤ 4 for any n up to ~10⁸⁰. Effectively constant for all practical purposes.

Proofs: Optimal Substructure

Proof Blocks Format On the quiz, you select correct steps and arrange them in order. Below are the correct steps for each algorithm.

Coin Change — Optimal Substructure

Claim: If C = {c₁, …, cₙ} is optimal for A cents, then C' = {c₂, …, cₙ} is optimal for A − c₁.

1Assume C is optimal for A (n coins), but C' is NOT optimal for A − c₁.
2∃ C'' making A − c₁ cents using j < n − 1 coins.
3Construct C''' = {c₁} ∪ C''. Makes A cents using j + 1 coins.
4j + 1 < (n−1) + 1 = n. C''' uses fewer coins than C for amount A.
5Contradiction: C''' is better than optimal C. ∴ C' is optimal for A − c₁. ∎
Distractors to Reject

• "We can split A into any two amounts and both are optimal" (FALSE)

• Steps proving the converse direction (combining sub-solutions)

Dijkstra's — Optimal Substructure

Claim: If P = {p₁, …, pₙ} is the shortest path p₁→pₙ, then P' = {p₁, …, pₙ₋₁} is shortest p₁→pₙ₋₁.

1Assume P is optimal from p₁ to pₙ, but P' is NOT optimal from p₁ to pₙ₋₁.
2∃ better path P'' from p₁ to pₙ₋₁ with cost(P'') < cost(P').
3Construct path P'' ∪ {pₙ}: follow P'' to pₙ₋₁, then take last edge to pₙ.
4cost(P'' ∪ {pₙ}) = cost(P'') + cₙ < cost(P') + cₙ = cost(P).
5Contradiction: new path to pₙ is cheaper than optimal P. ∴ P' is optimal. ∎
MST — Optimal Substructure

Claim: If T is MST and we remove edge e, the two subtrees T' and T'' are MSTs of their subgraphs.

1Assume T is optimal MST (cost c), but subtree T' is NOT optimal for its subgraph (cost c').
2∃ better sub-MST T''' with cost c''' < c'.
3Replace T' with T''' in the original: reconnect via edge e and T''.
4New tree has cost c − c' + c''' < c (since c''' < c').
5Contradiction: new spanning tree has lower cost than optimal T. ∴ T' is optimal. ∎
Activity Selection — Optimal Substructure

Claim: If A = {a₁, …, aₙ} is optimal, then A' = {a₂, …, aₙ} is optimal for time frame starting at f₁.

⚠ Subproblem = only activities starting at or after f₁ (finish time of a₁). Overlapping activities excluded.

1Assume A is optimal (n activities) but A' is NOT optimal for subproblem after f₁.
2∃ A'' for the subproblem with |A''| > |A'| = n − 1.
3Construct {a₁} ∪ A''. Feasible because all of A'' starts after f₁.
4|{a₁} ∪ A''| = 1 + |A''| > 1 + (n−1) = n = |A|.
5Contradiction: more activities than optimal A. ∴ A' is optimal. ∎
Fractional Knapsack — Optimal Substructure

Claim: If X = {x₁, …, xₙ} is optimal for capacity C, then X' = {x₂, …, xₙ} is optimal for C − x₁·w₁.

1Note V(X) = V(x₁) + V(X').
2Assume X is optimal for C but X' is NOT optimal for C − x₁·w₁.
3∃ X'' with V(X'') > V(X') that fills C − x₁·w₁.
4Construct {x₁} ∪ X'': value = V(x₁) + V(X'') > V(x₁) + V(X') = V(X).
5Contradiction: better than optimal X. ∴ X' is optimal. ∎

Proofs: Greedy Choice Property

Coin Change — Greedy Choice (US coins {25,10,5,1})

Claim: Some optimal solution contains the largest coin cᵢ where cᵢ ≤ A.

1Case: penny (A ≤ 4). Only pennies available — trivially true.
2Case: nickel (5 ≤ A ≤ 9). No nickel → ≥5 pennies. Replace 5 pennies with 1 nickel → fewer coins.
3Case: dime (10 ≤ A ≤ 24). No dime → only nickels & pennies. All sub-cases (≥2 nickels, 1 nickel + ≥5 pennies, ≥10 pennies) can be replaced by a dime.
4Case: quarter (A ≥ 25). No quarter → only dimes/nickels/pennies. Any combo ≥25¢ has a 25¢ subset replaceable by one quarter.
5In all cases, an optimal solution with the largest coin exists. ∎
Dijkstra's — Greedy Choice (Correctness by Induction)

Claim: When node v is made known, v.d = δ(s, v).

1Base: s.d = 0 = δ(s, s). ✓
2IH: First k known nodes have correct shortest paths.
3Node v is (k+1)th known. v.d = δ(s,u) + c(u,v) — minimum over all fringe nodes.
4Suppose ∃ better path: s→u'→v'→…→v with cost δ(s,u') + c(u',v') + Δ < v.d, where Δ ≥ 0.
5Since Δ ≥ 0: δ(s,u') + c(u',v') < v.d.
6But then v' has smaller PQ key than v → Dijkstra's would have selected v' first. Contradiction. ∎
Key: Why No Negative Edges

Step 4 requires Δ ≥ 0. With negative edges, the remaining path cost could be negative, breaking Step 5.

Prim's — Greedy Choice Property

Claim: The cheapest edge from known set to fringe is in some MST.

1Let e = (S, v) be cheapest edge from known to unknown. Assume e is NOT in any MST.
2Pick MST T. T already has a path from S to v (since T spans everything).
3Adding e to T creates a cycle. On this cycle, ∃ another edge e' crossing the same cut.
4Swap: T' = T − e' + e. Still a valid spanning tree.
5weight(T') = weight(T) − weight(e') + weight(e). Since e is cheapest cut edge: weight(e) ≤ weight(e').
6∴ weight(T') ≤ weight(T). T' is an MST containing e. Contradiction. ∎
Kruskal's — Greedy Choice Property

Claim: The globally cheapest edge not creating a cycle is in some MST.

1Let e = (u,v) be cheapest edge connecting two different components. Assume e ∉ any MST.
2Pick MST T. Adding e creates a cycle (T already connects u to v).
3On the cycle, ∃ edge e' connecting the same two components.
4Since Kruskal's processes edges in order: weight(e) ≤ weight(e').
5T' = T − e' + e has weight(T') ≤ weight(T). T' is MST containing e. Contradiction. ∎
Activity Selection — Greedy Choice Property

Claim: The activity with the earliest finish time (i₁) is in some optimal solution.

1Assume i₁ ∉ optimal solution O = {o₁, o₂, …, oₙ} (o₁ finishes first in O).
2i₁.f ≤ o₁.f (i₁ has earliest finish of ALL activities).
3o₂.s ≥ o₁.f (O is feasible → o₂ starts after o₁ finishes).
4Combining: i₁.f ≤ o₁.f ≤ o₂.s → i₁ compatible with o₂ and all subsequent.
5Exchange: O' = {i₁, o₂, …, oₙ}. |O'| = |O| = n. O' is optimal and contains i₁. Contradiction. ∎
Important

The exchange produces a solution of equal size, not strictly better. We're showing i₁ CAN be in an optimal solution, not that it improves it.

Fractional Knapsack — Greedy Choice Property

Claim: The item with highest ratio rₙ, taken maximally, is in some optimal solution.

1Assume best-ratio item iₙ NOT taken maximally in optimal O. Let δ = Max(iₙ.w, C) − oₙ > 0.
2∃ item j with oⱼ > 0 and rⱼ ≤ rₙ (knapsack full, some weight in a worse-ratio item).
3Transfer ε ≤ δ weight from item j to item n.
4Value change: −ε·rⱼ + ε·rₙ = ε(rₙ − rⱼ) ≥ 0.
5V(O') = V(O) + ε(rₙ − rⱼ) ≥ V(O). O' is optimal with more of iₙ. Contradiction. ∎
Distractors to Reject

• "The value strictly increases" — FALSE, it's ≥ 0, could be 0 if ratios equal.

• Steps that swap entire items (that's 0/1 knapsack logic).

• "Take the highest value item" (wrong greedy choice).

Pitfalls & Counterexamples

Coin Change

D = {1, 6, 10, 25}, A = 12. Greedy: 10+1+1 (3). Optimal: 6+6 (2). Proof fails at dime case.

Dijkstra's ≠ MST

Triangle: A-B(1), A-C(2), B-C(1). Dijkstra's from A: tree weight 3. MST weight: 2. Different trees.

0/1 Knapsack Greedy Fails

Max value: n=3, C=20 → profit 28.2 (not 31.5). Min weight: C=10, items (v=1,w=1)+(v=30,w=10) → profit 28 (not 30).

Activity Selection: Wrong Greedy Choices

Earliest start, shortest interval, and fewest conflicts all have known counterexamples. Only earliest finish works.

Bridge Crossing: Neither Strategy Always Wins

Must compare sₙ₋₁ + s₁ − 2s₂ at each step. Using only fastest-escorts or only slowest-together both have counterexamples.

General Rules to Remember

• Greedy ≠ always correct. Every algorithm needs both properties proven.

• Optimal substructure defines a specific decomposition, not arbitrary splitting.

• The exchange argument shows the greedy choice is in some optimal solution (≥ as good), not necessarily better.

• Dijkstra's requires no negative edges. Prim's doesn't care about negative edges.

• Know the PQ key difference: Dijkstra's = cumulative distance, Prim's = edge weight only.
